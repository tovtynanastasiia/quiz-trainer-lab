# Quiz Trainer - Інтерактивний Тренажер Слів

Сучасний веб-додаток для вивчення іноземних слів з використанням інтерактивних квізів. Побудований на **REACT** з **Supabase** для аутентифікації та зберігання даних.

## 🚀 Основні Можливості

### 📚 Управління Наборами Слів

- Створення власних наборів слів
- Додавання слів по одному або масово через текст
- Редагування існуючих слів
- Видалення слів та наборів

### 🎯 Інтерактивне Навчання

- **Режим "Навчання"** - для вивчення нових слів з адаптивними повтореннями
- **Режим "Точність"** - тренування уважності за 4 хвилини (20 слів)
- **Режим "Швидкість"** - розвиток швидкості набору за 4 хвилини
- **Режим "Флеш-картки"** - класичний метод вивчення без текстового вводу
- **Адаптивний алгоритм** - система shortMemory (5 кроків для вивчення слова)
- **Відображення правильних відповідей** при помилках
- **Затримка показу** недавно вивчених слів
- **Система повторень** - адаптивні інтервали для кращого запам'ятовування

### 📊 Відстеження Прогресу

- Збереження прогресу в базі даних
- Статистика спроб та правильних відповідей
- Персональні налаштування акаунту
- Детальний огляд вивчених слів
- **Прогрес-бари** для візуального відображення досягнень
- **Статистика по наборах** - окремі результати для кожного набору
- **Загальна статистика** - результати по всіх наборах разом

## 🗄 База Даних

### Структура таблиць:

- **sets** - набори слів користувачів
  - `id` (UUID) - унікальний ідентифікатор
  - `user_id` (UUID) - ідентифікатор користувача
  - `name` (TEXT) - назва набору
  - `description` (TEXT) - опис набору
  - `created_at` (TIMESTAMP) - дата створення

- **words** - слова в наборах
  - `id` (UUID) - унікальний ідентифікатор
  - `set_id` (UUID) - ідентифікатор набору
  - `hint` (TEXT) - підказка/слово
  - `answer` (TEXT) - відповідь/переклад
  - `created_at` (TIMESTAMP) - дата створення

- **user_progress** - прогрес вивчення слів
  - `id` (UUID) - унікальний ідентифікатор
  - `user_id` (UUID) - ідентифікатор користувача
  - `set_id` (UUID) - ідентифікатор набору
  - `word_id` (UUID) - ідентифікатор слова
  - `short_memory` (INTEGER) - рівень вивчення (0-5)
  - `updated_at` (TIMESTAMP) - дата оновлення

- **accuracy_results** - результати режиму точності
  - `id` (UUID) - унікальний ідентифікатор
  - `user_id` (UUID) - ідентифікатор користувача
  - `set_id` (UUID) - ідентифікатор набору
  - `correct_count` (INTEGER) - кількість правильних відповідей
  - `total_words` (INTEGER) - загальна кількість слів
  - `time_taken` (INTEGER) - час виконання в секундах
  - `created_at` (TIMESTAMP) - дата створення

- **speed_results** - результати режиму швидкості
  - `id` (UUID) - унікальний ідентифікатор
  - `user_id` (UUID) - ідентифікатор користувача
  - `set_id` (UUID) - ідентифікатор набору
  - `words_completed` (INTEGER) - кількість виконаних слів
  - `time_limit` (INTEGER) - ліміт часу (240 секунд)
  - `created_at` (TIMESTAMP) - дата створення

## 🏗 Архітектура Проекту

### Архітектурний патерн: Layered Architecture

```
┌─────────────────────────────────────────────────┐
│           Presentation Layer (pages/)           │
│         React Pages + React Router              │
└────────────────┬────────────────────────────────┘
                 │
┌────────────────┴────────────────────────────────┐
│         Component Layer (components/)           │
│         Reusable React Components               │
└────────────────┬────────────────────────────────┘
                 │
┌────────────────┴────────────────────────────────┐
│        Business Logic Layer (lib/)              │
│     quiz-logic/, quiz-data/, utilities          │
└────────────────┬────────────────────────────────┘
                 │
┌────────────────┴────────────────────────────────┐
│         Data Layer (Supabase)                   │
│      PostgreSQL + Supabase Auth + RLS           │
└─────────────────────────────────────────────────┘
```

### Принципи організації коду

- **Separation of Concerns** - чітке розділення відповідальностей
- **Component-based Architecture** - переиспользуемые компоненти
- **Single Responsibility Principle** - кожен модуль має одну відповідальність
- **Type Safety** - повна типізація через TypeScript
